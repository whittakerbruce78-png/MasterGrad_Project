# Локализация многостраничных статических сайтов

## Обзор
Этот скилл описывает процесс и предоставляет инструменты для автоматического пакетного редактирования и локализации HTML/PHP страниц на статических многостраничных сайтах.

Частая проблема при локализации — разная вложенность файлов (например, главная страница города лежит в `/khabarovsk/index.html`, а страницы районов в `/khabarovsk/centralniy-raion/index.html`). Этот скилл позволяет автоматически вычислять корректный относительный путь к общим ресурсам (`img/`, `css/`, `js/`) и безопасно заменять региональный контент.

## Зависимости
- Python 3.x
- Модули стандартной библиотеки: `os`, `re`, `json`

## Быстрый старт
Для применения локализации запустите скрипт автоматизации с конфигурационным файлом городов.

```bash
# Запуск скрипта локализации
python scripts/localize.py --config config/cities.json
```

## Шаблон скрипта локализации (scripts/localize.py)
Скрипт выполняет автоматический парсинг файлов, находит относительные пути на основе существующих ресурсов и делает точечные замены.

```python
import os
import re
import json

def get_nesting_level(file_path):
    """
    Вычисляет относительный префикс (например, '../' или '../../')
    на основе глубины залегания файла относительно корня.
    """
    rel_path = os.path.relpath(file_path, start='.')
    parts = rel_path.split(os.sep)
    depth = len(parts) - 1
    return '../' * depth if depth > 0 else './'

def replace_in_file(file_path, replacements):
    """
    Производит замены строк в файле на основе переданного словаря.
    """
    with open(file_path, 'r', encoding='utf-8') as f:
        content = f.read()

    modified = False
    for target, replacement in replacements.items():
        if target in content:
            content = content.replace(target, replacement)
            modified = True

    if modified:
        with open(file_path, 'w', encoding='utf-8') as f:
            f.write(content)
        print(f"[OK] Обновлен файл: {file_path}")
```

## Правила локализации
1. **Динамический путь к ресурсам**:
   Никогда не хардкодьте `../../img/` или `../img/`. Используйте функцию `get_nesting_level()` или считывайте относительный путь из уже подключенных ресурсов в HTML (например, из пути к `telegram.png` или `style.css`).
2. **Селектор городов**:
   Для кнопок выбора города используйте плейсхолдеры, которые заменяются на конкретный город текущей страницы:
   `г. [Название_Города] ▼`
3. **Безопасность автозамен**:
   Всегда делайте бэкап перед запуском регулярных выражений. Используйте точные строки замены (`replace()`), а не регулярные выражения, где это возможно, чтобы избежать случайной порчи верстки.

## Часто встречающиеся ошибки
* **Сбитые картинки**: Использование абсолютных путей `/img/...` вместо относительных `../img/...` приводит к поломке отображения на локальных стендах или GitHub Pages.
* **Потеря кодировки**: При перезаписи HTML/PHP файлов в Python всегда принудительно используйте `encoding='utf-8'`.
